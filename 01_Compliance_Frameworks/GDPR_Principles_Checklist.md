
# GDPR COMPLIANCE CHECKLIST (1-PAGE SUMMARY)

**Purpose:** To ensure compliance with the EU General Data Protection Regulation (GDPR).

| Category | Key Requirements | Status (✔ / ✖) | Notes |
|-----------|------------------|----------------|-------|
| **1. Lawful Basis** | Identify lawful basis for each processing activity (e.g., consent, contract). |  |  |
| **2. Consent** | Obtain explicit, informed consent where required. Maintain consent records. |  |  |
| **3. Data Minimization** | Collect only data necessary for the stated purpose. |  |  |
| **4. Transparency** | Maintain a clear and accessible privacy notice. |  |  |
| **5. Individual Rights** | Implement procedures for access, correction, deletion, and portability. |  |  |
| **6. Security** | Apply appropriate technical and organizational measures (encryption, access control). |  |  |
| **7. Data Breaches** | Establish process for reporting breaches within 72 hours to the DPA. |  |  |
| **8. Processor Agreements** | Ensure written contracts with processors include GDPR-compliant clauses. |  |  |
| **9. International Transfers** | Verify transfer mechanisms (SCCs, adequacy decision). |  |  |
| **10. Recordkeeping (Article 30)** | Maintain a record of processing activities (ROPA). |  |  |

✅ **Tip:** Review this checklist every 6–12 months or after major business changes.
