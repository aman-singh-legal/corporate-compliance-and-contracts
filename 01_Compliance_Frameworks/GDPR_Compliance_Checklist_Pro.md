## A. Foundations & Scope

| Area | Article(s) | Requirement | Status (✓/✗) | Notes |
|---|---|---|---|---|
| Territorial Scope | **Art. 3** | Confirm GDPR applies (EU establishment or targeting/monitoring EU individuals). |  |  |
| Material Scope | **Art. 2** | Confirm personal data/processing activities fall within GDPR scope (non-household, non-law-enforcement, etc.). |  |  |
| Definitions | **Art. 4** | Align internal definitions (personal data, special categories, controller, processor, profiling, etc.). |  |  |
| Representative | **Art. 27** | If not established in EU but subject to GDPR, appoint EU representative & document mandate. |  |  |
| Supervisory Authority | **Art. 51-59** | Identify lead supervisory authority (LSA) and cooperation channels. |  |  |

## B. Principles & Lawfulness

| Area | Article(s) | Requirement | Status (✓/✗) | Notes |
|---|---|---|---|---|
| Principles | **Art. 5(1)** | Implement principles: lawfulness, fairness, transparency, purpose limitation, data minimisation, accuracy, storage limitation, integrity/confidentiality. |  |  |
| Accountability | **Art. 5(2)** | Evidence compliance (policies, logs, assessments, training, audits). |  |  |
| Lawful Basis | **Art. 6(1)** | Map each processing purpose to one lawful basis (consent/contract/legal obligation/vital interests/public task/legitimate interests). |  |  |
| Legitimate Interests | **Art. 6(1)(f)** | Perform and document LIA (purpose test, necessity, balance). Provide opt-out where appropriate. |  |  |
| Consent – Quality | **Art. 7, 4(11)** | Freely given, specific, informed, unambiguous; easy withdrawal; granular; separate from T&Cs. |  |  |
| Children’s Consent | **Art. 8** | Obtain parental authorization where required; implement age-verification and consent records. |  |  |
| Special Categories | **Art. 9** | Identify processing of special data; verify Art. 9(2) condition; add safeguards. |  |  |
| Criminal Data | **Art. 10** | Process only under control of authority or if authorised by union/member-state law with safeguards. |  |  |

|---|---|---|---|---|

## D. Data Subject Rights

| Right | Article(s) | Requirement | Status (✓/✗) | Notes |
|---|---|---|---|---|
| Access | **Art. 15** | Provide copy of data; purposes, categories, recipients, retention, rights, sources, safeguards for transfers. 1 month deadline (extendable). |  |  |
| Rectification | **Art. 16** | Correct inaccurate data; complete incomplete data. |  |  |
| Erasure | **Art. 17** | Delete where grounds apply (withdrawal, no longer necessary, unlawful); manage exemptions. |  |  |
| Restriction | **Art. 18** | Mark data restricted; process only for limited purposes during verification/objection. |  |  |
| Portability | **Art. 20** | Provide data in structured, commonly used, machine-readable format for consent/contract basis and automated means. |  |  |
| Objection | **Art. 21** | Stop processing for legitimate interests/direct marketing unless compelling grounds documented. |  |  |
| Automated Decisions | **Art. 22** | Safeguards for solely automated decisions with legal/similar effects; enable human review; inform logic/impact. |  |  |

## Annex 1: ROPA (Art. 30) – Minimal Fields

**Controller Record (Art. 30(1))**  
- Name & contact of controller/representative/DPO  
- **Purposes** of processing  
- **Categories** of data subjects & personal data  
- **Recipients** (including third countries/international organisations)  
- **Transfers** to third countries + safeguards  
- **Retention** time limits (or criteria)  
- **Security** measures (general description)

**Processor Record (Art. 30(2))**  
- Name & contact of processor/each controller  
- **Categories** of processing carried out for each controller  
- **Transfers** to third countries + safeguards  
- **Security** measures (general description)

>  Keep ROPA current; align entries with notices, LIAs, DPIAs, DPAs, and data maps.

## Annex 2: DPIA – Trigger & Content

**Trigger (Art. 35(1))** – DPIA required when processing **likely to result in high risk**, including (non-exhaustive examples):  
- Systematic and extensive evaluation (including profiling) with significant effects  
- Large-scale processing of special categories/criminal data  
- Systematic monitoring of publicly accessible areas on a large scale

**Content (Art. 35(7))**  
- **Description** of processing, purposes, legitimate interests (if applicable)  
- **Necessity & proportionality** assessment  
- **Risks** to rights & freedoms of data subjects  
- **Measures** to address risks and demonstrate compliance (safeguards/security)  
- **Outcome** and residual risk; decision log (proceed/mitigate/consult SA per **Art. 36**)
